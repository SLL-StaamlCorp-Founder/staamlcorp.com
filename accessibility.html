Accessibility Statement

StaamlCorp is committed to ensuring digital accessibility for people of all abilities.

Our Commitment

StaamlCorp is dedicated to providing an inclusive online experience for all visitors, regardless of ability or technology. We strive to ensure that our website conforms to the Web Content Accessibility Guidelines (WCAG) 2.2 Level AA published by the World Wide Web Consortium (W3C).

Accessibility is an ongoing effort, and we continually work to improve the usability and accessibility of our website for all users.

Conformance Target

Our target is WCAG 2.2 Level AA conformance. WCAG 2.2 defines requirements for web content to be:

  • Perceivable: Information and user interface components must be presentable to users in ways they can perceive.
  • Operable: User interface components and navigation must be operable by all users.
  • Understandable: Information and the operation of the user interface must be understandable.
  • Robust: Content must be robust enough to be interpreted reliably by a wide variety of user agents, including assistive technologies.

Accessibility Features

We have implemented the following accessibility features across our website:

Structure and Navigation

  • Semantic HTML5 elements (header, nav, main, section, footer) for proper document structure
  • Logical heading hierarchy (h1 through h4) used consistently across all pages
  • Keyboard-navigable navigation menu, including mobile hamburger menu
  • Consistent navigation across all pages
  • Descriptive page titles for each page

Visual Design

  • Color contrast ratios that meet or exceed WCAG 2.2 AA requirements (minimum 4.5:1 for normal text, 3:1 for large text)
  • Information is not conveyed by color alone
  • Responsive design that adapts to different screen sizes and zoom levels up to 200%
  • Readable font sizes with relative units

Forms and Interactive Elements

  • All form fields have associated labels
  • Required fields are clearly identified
  • Form validation messages are accessible
  • Interactive elements have visible focus indicators
  • Buttons and links have descriptive text

Media and Content

  • Images include meaningful alt text where applicable
  • Decorative images are properly marked to be ignored by assistive technologies
  • Links are distinguishable from surrounding text
  • ARIA attributes used where appropriate to enhance screen reader compatibility

Cookie Consent

  • Cookie consent banner is keyboard navigable
  • Consent banner includes proper ARIA labels and roles
  • Screen reader announcements for consent banner

Known Limitations

Despite our best efforts, some areas of the website may not yet be fully accessible. Known limitations include:

  • Third-party content: Some third-party content or services embedded on our site may not fully conform to WCAG 2.2 AA standards. We work with our vendors to improve accessibility where possible.
  • Legacy content: Older content published before our current accessibility standards may not fully conform. We are working to update this content as part of our ongoing improvement efforts.
  • PDF documents: Some downloadable PDF documents may not be fully accessible. We are working to remediate these and provide accessible alternatives upon request.

Testing and Evaluation

We evaluate the accessibility of our website through:

  • Automated accessibility testing tools
  • Manual testing with keyboard navigation
  • Testing with screen readers (VoiceOver, NVDA)
  • Regular reviews against WCAG 2.2 Level AA criteria
  • User feedback and reported issues

Feedback

We welcome your feedback on the accessibility of our website. If you encounter any accessibility barriers or have suggestions for improvement, please contact us. We take all feedback seriously and will work to address issues promptly.

Report an Accessibility Issue

When reporting an issue, please include:

  • The web page URL where you encountered the issue
  • A description of the problem
  • The assistive technology you were using (if applicable)
  • Your browser and operating system

We aim to respond to accessibility feedback within 5 business days and resolve issues within 30 days where feasible.
